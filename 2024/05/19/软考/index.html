2.2 任务调度

任务调度方式

  • 不可抢占：主动放弃cpu使用权，只要该任务在运行，不可被别的打断
  • 可抢占：可以被打断

==不可重入函数：不能由超过一个函数所共享==

性能指标：

  • 响应时间：调度器为一个就绪任务进行上下文切换所需要的时间 + 任务在就绪链表中的等待时间
  • 周转时间：一个任务从周到到完成所经历的时间
  • 吞吐量：单位时间内完成任务的数量

任务调度算法

  • 先来先服务FCFS

    • 平均周转时间：==假如Ta<Tb<Tc，如果按Ta，Tb，Tc顺序执行，则 T = (3Ta+2Tb+Tc)/3==
    • 短作业优先算法：选择执行时间短的任务先执行

  • 时间片轮转RR

  • 优先级算法

    • 可抢占
    • 不可抢占

任务间通信

  • 低级：信号量、信号
  • 高级：
    • 共享内存，==需要额外的同步机制==

3 存储管理

  • 分区存储：把程序整个直接调入到内存中执行

    • 静态
    • 动态

  • 页式存储：将程序分成若干个页，每次调取某页到内存中某块去执行，==逻辑地址== = 页号+页内地址(页大小)

    • 页号，页内地址，一个页号对应个块号
    • 块号，物理地址 = 快号 + 页内地址
    • ==求物理地址==：
      • 先根据页号找到物理快号，再用 块号+页内地址 就是对应的物理地址
      • 逻辑地址/页大小 = 页号，取整找到对应块号，逻辑地址%页大小 = m，取余找到m，物理地址 = 块号*页大小+m
    • ==优点：==利用率高、碎片小、分配和管理简单
    • ==缺点：==增加了系统开销，可能产生抖动现象

  • 段式存储：==逻辑地址== = 段号+段内地址（位移量）

    • ==求物理地址==：根据段号找到段长，段表= 段长+基址，物理地址 = 基址 + 位移量
    • ==优点：==多道程序内存共享，各段程序不干扰
    • ==缺点：==利用率低，内存碎片浪费大（外碎片）

  • 段页式存储：==逻辑地址== =段号+页号+页内地址

    • ==优点：==空间浪费小、存储共享容易，动态链接
    • ==缺点：==增加复杂性和开销，执行速度低

  • 虚拟存储

    • 页面置换算法：存储块不够时，需要淘汰某个页面：淘汰访问过的，但是未被修改的页面

  • 磁盘结构及调度算法

    • ==读取磁盘数据时间==：
      • 磁道时间
      • 块（扇区）时间，即旋转延迟时间
      • 传输时间
    • 非格式化容量：面数 x（磁道数/面）x 内圆周长 x 最大位密度
    • 格式化容量：面数 x（磁道数/面）x （扇区数/道） x （字节数/扇区）
    • ==题型==
      • 求处理最长时间：
        • 先算旋转一个物理块时间 t1= 磁盘旋转周期/物理块个数 ，处理一个物理块时间 t2
        • T = (t1+t2) + 【 (t1*(总块数-1)+t1+t2) * （总块数-1）】
      • 求最短时间：
        • T = （t1+t2）* 总块数

    • 缺点：
      • 加密速度慢

      • 条件覆盖CC，每个逻辑条件的可能值
      • 条件判定覆盖C/DC，同时满足判定覆盖和条件覆盖
      • 条件组合覆盖MCC，每条路径至少被覆盖一次
      • 修正的条件判定覆盖MC/DC，同时满足判定覆盖和条件覆盖，并且满足两次计算
        • 条件C所在判定内的所有条件，除条件C外，其他条件的取值完全相同
        • 条件C取值相反
        • 判定的计算结果相反

调试与测试的区别

  • 测试是找出存在的错误；调式是定位错误并修正错误
  • 调式在测试之后
  • 测试在已知的条件下，有预支结果；调式在未知的条件下，没有预支结果
  • 测试可以事先设计，进度确定；调试不能描述过程或持续时间
